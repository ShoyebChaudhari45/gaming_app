1. Project Overview & Statistics

Game App is a full-featured Android gaming application built in Java with Retrofit for REST API integration. It provides user authentication, real-time wallet management (deposit & withdrawal), a bidding/lottery system with multiple game types, and complete transaction history — all wrapped in a clean Material Design UI.

6. Authentication Module

6.1 LoginActivity

  • Auto-login: If SessionManager.isLoggedIn() is true, skips to home immediately
  • Validation: Mobile must match regex ^[6-9]\d{9}$ (Indian 10-digit), password must not be empty
  • API Call: POST /user/login with LoginRequest(mobile_no, password)
  • On success: saves token, user name, mobile, email, user type, fetches full details (balance, QR)
  • Retry mechanism: If user-details fetch fails, retries up to 3 times with delays [500ms, 1500ms, 3000ms]
  • Security: Shows generic "Invalid mobile or password" on failure (prevents enumeration)
  • Panel Access: Button to open web panel at lottery.durwankurgroup.com/panel/login

6.2 RegisterActivity

  • Fields: Name, Mobile, Email, Password, Referral Code (optional), User Type (Customer/Employee radio)
  • Validations: Name required, 10-digit mobile, valid email pattern, min 4-char password, user type selection
  • Backend error handling: Parses error JSON — shows "Email already exists" or "Mobile already exists"
  • Password reveal: On validation failure, switches to visible password to show the error

6.3 ForgotPasswordActivity

  • Flow: Enter email → POST /user/forgot-password → OTP sent → redirect to ResetPasswordActivity
  • Validation: Checks empty + valid email pattern using Patterns.EMAIL_ADDRESS

6.4 ResetPasswordActivity

  • Fields: 6-digit OTP, New Password (min 6 chars), Confirm Password
  • API: POST /user/reset-password with email, OTP, new_password, confirmation
  • Resend OTP: POST /user/resend-otp with email
  • On success: Clears task stack, redirects to LoginActivity

6.5 ChangePasswordActivity

  • Email: Auto-fetched from SessionManager (not user input)
  • API: POST /user/change-password with Bearer token
  • On success: Shows AlertDialog → "Login Again" button → Logout → Login screen

8. Game & Bidding System

8.1 Game Taps (HomeActivity)

Games are loaded from GET /taps. Each game has open/close taps with statuses: running, open, upcoming, closed. Games are sorted by priority for better UX.

8.2 GameTypesActivity

Displays game types (Open, Jodi, SP, DP, Cycle, Patte) in a 2-column grid using GridLayoutManager. Loaded from GET /games. Each type has a name and image.

8.3 BidActivity — Placing Bids

  • Open/Close selection: Cards toggle between open and close sessions; unavailable sessions are grayed out (alpha 0.4)
  • Auto-format digits: TextWatcher formats input based on game type — Jodi/Cycle: 2-digit blocks, Patte: 3-digit blocks, Open/SP/DP: single digits, separated by =
  • Duplicate detection: LinkedHashSet prevents duplicate digit blocks
  • Validation: Checks digit format, points > 0, session availability
  • Confirmation dialog: Shows Game, Type, Digit, Price before submitting
  • API: POST /lottery with timeId, type, digit, price
  • Success animation: Lottie tick animation + success sound + auto-dismiss after 4s
  • Balance refresh: Wallet balance is refreshed from API after successful bid

8.4 BidHistoryActivity

Date-range filtered bid history with DatePickerDialog. API: GET /lottery/list?start_date=&end_date=. Default range: last 7 days. Shows empty state when no bids found.

8.5 Game Rates & Starline

  • GameRatesActivity: Fetches from GET /gamerate, displays name + price-digit rate
  • StarlineActivity: Fetches from GET /starline_rate, displays starline-specific rates

9. Wallet System

9.1 AddPointsActivity (Deposit)

  • QR Code: Loaded via Glide from SessionManager URL with disk caching
  • Quick prices: Grid of preset amounts from GET /prices, auto-fills amount field on tap
  • File upload: User selects payment proof image → converted via FileUtils.getFileFromUri()
  • API: POST /wallet/deposit as multipart (amount + payment_proof file)
  • Balance refresh: Button with 360° rotation animation

9.2 WithdrawActivity

  • Validation: Minimum ₹100, cannot exceed balance, proof image required
  • API: POST /wallet/withdraw as multipart (amount + payment_mode "upi" + payment_proof)
  • Balance not deducted: Balance deduction happens only after backend approval (pending status)

9.3 WalletStatementActivity

Displays transaction history from GET /wallet/statement using WalletStatementAdapter. Shows balance, add funds & withdraw shortcuts.

10. Profile Management

  • View mode: Name, Email (disabled), Mobile (always disabled), User Type badge
  • Edit mode: Toggle via "Edit Profile" button → enables Name & Email fields, button changes to "Submit Request"
  • API: POST /profile with name, email; Fetch: GET /user/details
  • Logout: AlertDialog confirmation → clear session → LoginActivity

13. API Layer

13.1 ApiClient.java — Retrofit Singleton

  • Base URL: https://lottery.durwankurgroup.com/api/
  • OkHttp timeouts: Connect: 15s, Read: 15s, Write: 15s
  • Converter: GsonConverterFactory for automatic JSON ↔ Java serialization
  • Pattern: Lazy singleton — instance created on first call

13.2 ApiService.java — Interface

Defines all 22 API endpoints using Retrofit annotations (@GET, @POST, @DELETE, @Multipart). All secured endpoints accept @Header("Authorization") for Bearer token.

18. Security Implementation

🔐 Authentication

  • Bearer token in every secured API call
  • Token stored in SharedPreferences
  • Auto-logout on session expiry

🛡️ Network

  • HTTPS only (cleartext disabled)
  • Network security config in manifest
  • 15-second timeouts prevent hang

✅ Input Validation

  • Mobile: Indian 10-digit regex
  • Email: Patterns.EMAIL_ADDRESS
  • Password: min length enforcement
  • Amount: min ₹100 for withdraw

🗑️ Account Deletion

  • Double AlertDialog confirmation
  • DELETE API call with token
  • Full session clear on delete
  • Redirects to login with clear task

19. Role-Based Access

The app supports two user roles: Customer and Employee.
